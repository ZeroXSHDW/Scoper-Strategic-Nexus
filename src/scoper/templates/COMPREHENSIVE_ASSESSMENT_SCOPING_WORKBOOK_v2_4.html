How to Use This Workbook

1. Start with business context

Complete the executive summary, project owner details, business objectives, compliance drivers, target dates, and section completion matrix.

2. Route to sections

Select the relevant assessment types, then complete only the matching technical sections. Mark excluded sections as not applicable with rationale.

3. Capture technical boundaries

For each selected section, identify exact targets, owners, data classification, environments, credentials, access method, restrictions, and expected deliverables.

4. Map compliance

Use the compliance appendices to confirm required testing types, frequency, evidence, and audit-ready outputs.

5. Convert to SOW

Use SOW handoff fields to convert scope into statement of work language, client dependencies, assumptions, exclusions, and acceptance criteria.

6. Sign off safely

Confirm authorization, emergency stop, evidence handling, retest expectations, and change control before work begins.
